## 1. Analysis Summary

**Analysis Date:** 2026-04-30
**C# Source Location:** `Emby.Server/`, `MediaBrowser.*/`
**Analysis Method:** Directory structure review, key file analysis

### Key Findings

1. **Original plan covered core components well** — Library, Session, User, System APIs

   - Channel management
   - LiveTV/EmbyTV system
   - HttpClientManager for HTTP reuse
   - Playlist service
   - Display preferences
   - ISO/disc mounting

4. **DLNA implementation is partial** — Basic UPnP discovery only

### 3.4 Authentication and Security

#### Password Hashing

**Location:** `MediaBrowser.Server.Implementations/Auth`

**Algorithm:** PBKDF2-SHA256
**Iterations:** 100,000
**Salt:** 4 bytes (32 bits)
**Hash:** 4 bytes (32 bits) output stored as 16-char hex

```csharp
// C# Implementation
using (var provider = new Rfc2898DeriveBytes(password, salt, iterations, HashAlgorithmName.SHA256))
{
    byte[] hash = provider.GetBytes(4);  // Only 4 bytes!
    // Stored as hex: 8 characters
}
```

**Critical:** The Go implementation must match exactly:
- Same iterations count
- Same salt length
- Same output length
- Same hash algorithm

#### API Key Authentication

**Headers:**
- `X-Emby-Token` — API key or session token
- `Authorization` — Bearer token format

#### Session Tokens

- 14-day expiry
- Stored in `devices` table
- Refresh on activity

## 5. Database Schema Notes

### Key Tables

#### TypedBaseItems

Main table for all media items.

| Column | Type | Purpose |
|--------|------|---------|
| Id | TEXT (GUID) | Primary key |
| Name | TEXT | Display name |
| Path | TEXT | File path |
| ItemType | TEXT | Movie, Episode, Song, etc. |
| ParentId | TEXT | Parent folder ID |
| PremiereDate | DATETIME | Release date |
| ProductionYear | INT | Year |
| PrimaryImageId | TEXT | Image ID |
| Data | BLOB | Serialized metadata |
| SortName | TEXT | For sorting |

#### Users

| Column | Type | Purpose |
|--------|------|---------|
| Id | TEXT (GUID) | Primary key |
| Username | TEXT | Login name |
| Password | TEXT | PBKDF2 hash |
| PasswordSalt | TEXT | Salt (hex) |
| PrimaryImageId | TEXT | Avatar ID |
| Configuration | BLOB | User settings |
| LastActivityDate | DATETIME | Last login |

### Indexes

```sql
CREATE INDEX idx_items_type ON TypedBaseItems(ItemType);
CREATE INDEX idx_items_parent ON TypedBaseItems(ParentId);
CREATE INDEX idx_users_username ON Users(Username);
```
