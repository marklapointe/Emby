## 1. API Overview

### 1.1 Base URL

```
http://{host}:8091/emby
```

For production deployments behind a reverse proxy:
```
https://{host}/emby
```

### 1.2 Authentication

#### API Key Authentication

**Header:** `X-Emby-Token: {api_key}`

#### Session Token Authentication

**Header:** `X-Emby-Token: {session_token}`

#### Login Request

```http
POST /emby/Users/{userId}/Authenticate
Content-Type: application/json

{
  "Username": "string",
  "Pw": "string"
}
```

**Response:**
```json
{
  "User": {
    "Id": "string",
    "Name": "string"
  },
  "AccessToken": "string",
  "ServerId": "string"
}
```

### 1.3 Response Format

#### JSON Response

```http
GET /emby/Items
Accept: application/json
```

**Response:**
```json
{
  "Items": [
    {
      "Id": "string",
      "Name": "string",
      "Type": "Movie",
      "MediaType": "Video"
    }
  ],
  "TotalRecordCount": 100,
  "StartIndex": 0
}
```

#### Image Response

```http
GET /emby/Items/{id}/Images/Primary
Accept: image/jpeg
```

Returns binary image data with appropriate `Content-Type` header.

### 1.4 Error Handling

**Error Response:**
```json
{
  "error": "string",
  "message": "string",
  "code": "string"
}
```

**HTTP Status Codes:**

| Code | Meaning |
|------|---------|
| 200 | Success |
| 201 | Created |
| 204 | No Content |
| 400 | Bad Request |
| 401 | Unauthorized |
| 403 | Forbidden |
| 404 | Not Found |
| 500 | Internal Server Error |

## 2. Authentication API

### 2.1 Authenticate

**POST** `/emby/Users/{userId}/Authenticate`

Authenticate a user.

**Request:**
```json
{
  "Username": "string",
  " Pw": "string"
}
```

**Response:**
```json
{
  "User": {
    "Id": "string",
    "Name": "string",
    "PrimaryImageTag": "string"
  },
  "AccessToken": "string",
  "ServerId": "string"
}
```

### 2.2 Get Keys

**GET** `/emby/Auth/Keys`

Get active API keys.

**Response:**
```json
{
  "Items": [
    {
      "DeviceId": "string",
      "DeviceName": "string",
      "DateCreated": "2026-04-30T00:00:00Z",
      "AccessToken": "string"
    }
  ]
}
```

### 2.3 Remove Key

**DELETE** `/emby/Auth/Keys`

Remove an API key.

**Query Parameters:**
- `deviceId` (string): Device ID to remove

## 3. Library API

### 3.1 Get Libraries

**GET** `/emby/Library`

Get configured libraries.

**Response:**
```json
{
  "Items": [
    {
      "Id": "string",
      "Name": "string",
      "CollectionType": "movies",
      "ImageTags": {}
    }
  ]
}
```

### 3.2 Get Items

**GET** `/emby/Items`

Browse items with filtering.

**Query Parameters:**
| Parameter | Type | Description |
|-----------|------|-------------|
| `parentId` | string | Parent folder ID |
| `includeItemTypes` | string | Filter by type (Movie, Episode, etc.) |
| `startIndex` | int | Pagination start |
| `limit` | int | Page size |
| `sortBy` | string | Sort field |
| `sortOrder` | string | Ascending/Descending |
| `searchTerm` | string | Search term |
| `genres` | string | Genre filter (comma-separated) |
| `years` | string | Year filter (comma-separated) |
| `enableUserData` | bool | Include user data |

**Response:**
```json
{
  "Items": [
    {
      "Id": "string",
      "Name": "string",
      "Type": "Movie",
      "MediaType": "Video",
      "ProductionYear": 2024,
      "PremiereDate": "2024-01-15",
      "CommunityRating": 8.5,
      "RunTimeTicks": 72000000000,
      "ParentId": "string"
    }
  ],
  "TotalRecordCount": 100,
  "StartIndex": 0
}
```

### 3.3 Get Item

**GET** `/emby/Items/{id}`

Get item details.

**Response:**
```json
{
  "Id": "string",
  "Name": "string",
  "Type": "Movie",
  "Overview": "string",
  "OfficialRating": "R",
  "CommunityRating": 8.5,
  "ProductionYear": 2024,
  "PremiereDate": "2024-01-15",
  "Genres": ["Action", "Adventure"],
  "Studios": [{"Name": "Studio Name"}],
  "MediaSources": [
    {
      "Id": "string",
      "Protocol": "File",
      "Path": "/path/to/file.mkv",
      "Container": "mkv",
      "Size": 15000000000,
      "Streams": [
        {
          "Type": "Video",
          "Codec": "h264",
          "Width": 1920,
          "Height": 1080,
          "BitRate": 8000000
        },
        {
          "Type": "Audio",
          "Codec": "aac",
          "Channels": 6,
          "BitRate": 384000
        }
      ]
    }
  ]
}
```

### 3.4 Refresh Item

**POST** `/emby/Items/{id}/Refresh`

Refresh metadata for an item.

**Query Parameters:**
| Parameter | Type | Description |
|-----------|------|-------------|
| `metadataCountryCode` | string | Country for metadata |
| `metadataLanguageCode` | string | Language code |

### 3.5 Delete Item

**DELETE** `/emby/Items/{id}`

Delete an item.

### 3.6 Get Genres

**GET** `/emby/Genres`

Get all genres.

**Response:**
```json
{
  "Items": [
    {"Name": "Action", "Id": "string"},
    {"Name": "Comedy", "Id": "string"}
  ]
}
```

### 3.7 Get Studios

**GET** `/emby/Studios`

Get all studios.

### 3.8 Get Years

**GET** `/emby/Years`

Get all production years.

## 4. Playback API

### 4.1 Get Video Stream

**GET** `/emby/Videos/{id}/stream`

Get video stream URL or direct stream.

**Query Parameters:**
| Parameter | Type | Description |
|-----------|------|-------------|
| `static` | bool | Return direct file path |
| `mediaSourceId` | string | Media source ID |
| `deviceId` | string | Device ID |
| `api_key` | string | API key |

**Response:**
```json
{
  "Protocol": "File",
  "Id": "string",
  "Path": "/path/to/file.mkv",
  "MimeType": "video/x-matroska",
  "Container": "mkv"
}
```

### 4.2 Get Subtitle

**GET** `/emby/Videos/{id}/{streamIndex}/subtitles/{subtitleIndex}/stream`

Get subtitle stream.

**Response:**
Returns subtitle file (SRT, ASS, etc.)

### 4.3 Get Subtitle Format

**GET** `/emby/Videos/{id}/{streamIndex}/subtitles/{subtitleIndex}/stream.vtt`

Get subtitle converted to WebVTT format.

### 4.4 Start Live TV

**POST** `/emby/LiveTv/LiveStream`

Start a live TV stream.

**Request:**
```json
{
  "ChannelId": "string",
  "DeviceId": "string"
}
```

**Response:**
```json
{
  "MediaSource": {
    "Id": "string",
    "Protocol": "Http",
    "Path": "http://stream-url",
    "Container": "ts"
  }
}
```

## 5. Session API

**Response:**
```json
{
  "Items": [
    {
      "Id": "string",
      "DeviceId": "string",
      "DeviceName": "string",
      "UserId": "string",
      "UserName": "string",
      "ApplicationVersion": "1.0.0",
      "NowPlayingItem": {
        "Id": "string",
        "Name": "string"
      },
      "PlayState": {
        "PositionTicks": 72000000000,
        "CanSeek": true,
        "IsPaused": false
      }
    }
  ]
}
```

### 5.2 Create Session

**POST** `/emby/Sessions`

Create a new session.

### 5.3 Ping Session

**POST** `/emby/Sessions/{id}/Ping`

Keep session alive.

**Request:**
```json
{
  "PositionTicks": 72000000000
}
```

### 5.4 End Session

**POST** `/emby/Sessions/{id}/Logout`

End a session.

### 5.5 Send Play Command

**POST** `/emby/Sessions/{id}/Playing`

Send play command to session.

**Request:**
```json
{
  "ItemIds": ["string"],
  "PlayCommand": "PlayNow",
  "StartPositionTicks": 0
}
```

### 5.6 Send Progress Command

**POST** `/emby/Sessions/{id}/Playing/Progress`

Send playback progress.

**Request:**
```json
{
  "PositionTicks": 72000000000,
  "IsPaused": false,
  "EventName": "timeupdate"
}
```

## 7. Image API

### 7.1 Get Image

**GET** `/emby/Items/{id}/Images/{type}`

Get item image.

**Path Parameters:**
- `type`: Primary, Backdrop, Logo, Thumb, Chapter

**Query Parameters:**
| Parameter | Type | Description |
|-----------|------|-------------|
| `maxWidth` | int | Maximum width |
| `maxHeight` | int | Maximum height |
| `quality` | int | Quality 1-100 |
| `tag` | string | Image tag for caching |

**Response:**
Binary image data (JPEG, PNG, WebP)

### 7.2 Upload Image

**POST** `/emby/Items/{id}/Images/{type}`

Upload item image.

**Body:** Binary image data

### 7.3 Delete Image

**DELETE** `/emby/Items/{id}/Images/{type}/{index}`

Delete an image.

## 11. WebSocket API

### 11.1 Connection

**WebSocket:** `ws://{host}:8092/embywebsocket`

**Authentication:** Send `Login` message after connect

### 11.2 Message Format

```json
{
  "MessageId": "string",
  "MessageType": "string",
  "Data": {}
}
```

### 11.3 Message Types

| Type | Direction | Description |
|------|----------|-------------|
| `Login` | C→S | Authenticate |
| `LoginResult` | S→C | Auth result |
| `SessionUpdates` | S→C | Session info updates |
| `LibraryChanged` | S→C | Library modification |
| `UserDataChanged` | S→C | User data update |
| `Play` | C→S | Start playback |
| `PlayState` | C→S | Playback state |
| `PlayStateUpdate` | S→C | Playback update |
| `GeneralCommand` | C→S | Send command |
| `ForceKeepAlive` | S→C | Keep-alive ping |

### 11.4 Login Message

```json
{
  "MessageType": "Login",
  "Data": {
    "Username": "string",
    "Pw": "string"
  }
}
```

### 11.5 Play Command

```json
{
  "MessageType": "Play",
  "Data": {
    "ItemIds": ["string"],
    "PlayCommand": "PlayNow",
    "StartPositionTicks": 0
  }
}
```
